# AFCE Pseudo Language (APL) — документация

## Введение

**AFCE Pseudo Language (APL)** — это текстовый псевдоязык для описания блок-схем алгоритмов в стиле PlantUML. 
Он предназначен для быстрого создания блок-схем, соответствующих **ГОСТ 19.701-90**, 
без необходимости рисовать блоки мышью.

Язык преобразуется во внутренний XML-формат AFCE, который затем отображается как 
готовая блок-схема.

## Быстрый старт

```apl
@start_scheme
  process "Начало работы";
  input a, b;
  assign result = a + b;
  output result;
@stop_scheme
```

> Вставьте этот код в редакторе через меню *File → Import pseudo-language...* 
> и нажмите OK.

## Синтаксис языка

### Общие правила

- Каждый оператор завершается точкой с запятой `;`
- Блоки кода (тела условий, циклов) заключаются в фигурные скобки `{ }`
- Комментарии начинаются с `//` и действуют до конца строки
- Ключевые слова и названия блоков нечувствительны к регистру
- Строковые литералы заключаются в двойные кавычки `"..."`

### Структура файла

```
@start_scheme
  // операторы блок-схемы
@stop_scheme
```

Маркер `@start_scheme` опционален — если его нет, парсер обработает код как есть.

## Типы блоков (соответствие ГОСТ 19.701-90)

### 1. Процесс (действие) — прямоугольник

Отображает **функцию обработки данных** (операцию или группу операций).

```
process "текст действия";
```

ГОСТ: п. 3.2.1.1 — *Процесс*

**Пример:**
```apl
process "Вычислить сумму элементов";
```

![process]

---

### 2. Предопределённый процесс (вызов подпрограммы) — прямоугольник с полосками

Отображает вызов подпрограммы, функции или внешнего модуля.

```
call "имя_функции()";
```

ГОСТ: п. 3.2.2.1 — *Предопределенный процесс*

**Пример:**
```apl
call "sortArray(arr, n)";
```

---

### 3. Присваивание — прямоугольник с `:=`

Отображает операцию присваивания значения переменной.

```
assign переменная = выражение;
```

**Пример:**
```apl
assign result = 0;
assign i = i + 1;
assign ptr = (Node*)malloc(sizeof(Node));
```

На схеме отображается как `result := 0`.

---

### 4. Ввод данных — параллелограмм

Отображает операцию ввода данных (например, с клавиатуры).

```
input переменные;
```

ГОСТ: п. 3.1.2.5 — *Ручной ввод*

**Пример:**
```apl
input a, b, c;
```

---

### 5. Вывод данных — параллелограмм

Отображает операцию вывода данных (например, на экран).

```
output данные;
```

ГОСТ: п. 3.1.2.8 — *Дисплей*

**Пример:**
```apl
output "Результат: " + result;
output x, y, z;
```

---

### 6. Ветвление (условие) — ромб

Отображает **принятие решения** с двумя ветвями: Yes (Да) и No (Нет).

```
if (условие) {
    // действия, если условие истинно
} else {
    // действия, если условие ложно
}
```

Ветка `else` опциональна.

ГОСТ: п. 3.2.2.4 — *Решение*

**Пример:**
```apl
if (x > 0) {
    process "x положительный";
    assign count = count + 1;
} else {
    process "x неположительный";
}
```

**Пример с else-if:**
```apl
if (score >= 90) {
    process "Оценка: отлично";
} else if (score >= 70) {
    process "Оценка: хорошо";
} else {
    process "Оценка: удовлетворительно";
}
```

![if]

### 7. Цикл с предусловием — ромб с возвратом

Отображает цикл **while** — тело выполняется, пока условие истинно.

```
while (условие) {
    // тело цикла
}
```

ГОСТ: п. 3.2.2.6 — *Граница цикла* (предусловие)

**Пример:**
```apl
assign i = 0;
while (i < 10) {
    process "Обработка элемента";
    assign i = i + 1;
}
```

![while]

---

### 8. Цикл с постусловием — ромб с возвратом

Отображает цикл **do-while** — тело выполняется хотя бы один раз, 
затем проверяется условие.

```
do {
    // тело цикла
} while (условие);
```

ГОСТ: п. 3.2.2.6 — *Граница цикла* (постусловие)

**Пример:**
```apl
do {
    process "Чтение строки";
    input line;
} while (line != NULL);
```

![dowhile]

---

### 9. Цикл for — шестиугольник

Специальная форма цикла с инициализацией, условием и инкрементом.

```
for (инициализация; условие; инкремент) {
    // тело цикла
}
```

Три секции в скобках разделяются точкой с запятой.

**Пример:**
```apl
for (i = 0; i < n; i++) {
    process "Обработка";
}
```

### 10. Комментарий (пояснение) — скобка с пунктиром

Отображает **пояснительный комментарий** к блоку схемы.
Используется, когда текст не помещается в блоке или когда нужно пояснить ветвление.

```
comment "текст комментария";
```

ГОСТ: п. 3.4.3 — *Комментарий*

Комментарий прикрепляется к ближайшему не-comment блоку пунктирной линией
и отображается в виде квадратной скобки `[` рядом с блоком.

Графическое представление по ГОСТ:
```
[Блок] - - - -[ Текст комментария ]
```

Форма символа — открытый прямоугольник без левой стороны (фигура в виде буквы `[`).
Пунктирная линия связывает скобку с поясняемым блоком.
Текст размещается внутри или рядом со скобкой.

#### Правила позиционирования

- **Левый комментарий** (в `if` / `while` левой ветке, или при чередовании в центральном потоке) — текст и скобка размещаются **слева** от блока. Ветка автоматически расширяется влево, чтобы комментарий не обрезался.
- **Правый комментарий** (в `else` / правой ветке, или при чередовании в центральном потоке) — текст и скобка размещаются **справа** от блока. Ветка расширяется вправо.
- **Внутренние комментарии** (внутри веток `if` / `else`) разрывают соединительную линию потока: линия рисуется сегментами только между блоками, не сплошной через всю ветку. Это делает схему компактнее и чётко отделяет пояснения от потока управления.

**Пример:**
```apl
process "Действие с длинным описанием";
comment "Пояснение: это действие ...";
```

## Примеры

### Пример 1: Вычисление факториала

```apl
@start_scheme
  input n;
  assign result = 1;
  assign i = 1;
  
  if (n < 0) {
      output "Ошибка: отрицательное число";
  } else {
      while (i <= n) {
          assign result = result * i;
          assign i = i + 1;
      }
      output "Факториал: " + result;
  }
@stop_scheme
```

### Пример 2: Сортировка пузырьком

```apl
@start_scheme
  input arr, n;
  assign i = 0;
  
  while (i < n - 1) {
      assign j = 0;
      while (j < n - i - 1) {
          if (arr[j] > arr[j + 1]) {
              assign temp = arr[j];
              assign arr[j] = arr[j + 1];
              assign arr[j + 1] = temp;
          }
          assign j = j + 1;
      }
      assign i = i + 1;
  }
  
  output "Массив отсортирован";
@stop_scheme
```

### Пример 3: Удаление последнего элемента из списка (функция `dropLastUnit`)

```apl
@start_scheme
  process "Проверка: collection == NULL?";
  
  if (*collection == NULL) {
      process "return (коллекция пуста)";
  } else if ((*collection)->nextDelivery == NULL) {
      process "Освободить (*collection)->delivery";
      process "Освободить *collection";
      process "*collection = NULL";
      process "return";
  } else {
      process "temp = *collection";
      
      while (temp->nextDelivery->nextDelivery != NULL) {
          process "temp = temp->nextDelivery";
      }
      
      process "Освободить temp->nextDelivery->delivery";
      process "Освободить temp->nextDelivery";
      process "temp->nextDelivery = NULL";
  }
@stop_scheme
```

### 11. Данные (Data) — параллелограмм (наклон вправо)

Отображает **данные** в общем виде, носитель не определён.

```
data "текст";
```

ГОСТ: п. 3.1.1.1 — *Данные*

**Пример:**
```apl
data "Исходные данные";
```

---

### 12. Запоминаемые данные (Stored Data) — шестиугольник с вогнутыми сторонами

Отображает **хранимые данные**, носитель не определён.

```
stored_data "текст";
```

ГОСТ: п. 3.1.1.2 — *Запоминаемые данные*

**Пример:**
```apl
stored_data "Файл с данными";
```

---

### 13. Документ (Document) — прямоугольник с волнистым низом

Отображает данные на носителе в удобочитаемой форме.

```
document "текст";
```

ГОСТ: п. 3.1.2.4 — *Документ*

**Пример:**
```apl
document "Отчёт";
```

### 14. Ручной ввод (Manual Input) — трапеция со скошенным верхом

Отображает данные, вводимые вручную (клавиатура, переключатели).

```
manual_input "текст";
```

ГОСТ: п. 3.1.2.5 — *Ручной ввод*

**Пример:**
```apl
manual_input "Введите пароль";
```

ГОСТ: п. 3.2.2.2 — *Ручная операция*

```
parallel "текст";
```

ГОСТ: п. 3.2.2.5 — *Параллельные действия*

**Пример:**

ГОСТ: п. 3.4.1 — *Соединитель*

## Расширение языка

Язык спроектирован для лёгкого расширения. 
Чтобы добавить новый тип блока, необходимо:

1. Добавить распознавание ключевого слова в метод `parseStatement()` 
   в файле `pseudolangparser.cpp`
2. Создать соответствующий XML-элемент с нужными атрибутами
3. Убедиться, что элемент отображается в `zvflowchart.cpp` (метод `QBlock::paint()`)

## Отличия от PlantUML

AFCE Pseudo Language вдохновлён PlantUML, но имеет отличия:

| Характеристика          | PlantUML             | AFCE APL                  |
|-------------------------|----------------------|---------------------------|
| Маркеры начала/конца    | `@startuml`/`@enduml` | `@start_scheme`/`@stop_scheme` |
| Разделитель операторов  | `;` опционально      | `;` обязателен            |
| Ветвление               | `if (...) then/else/endif` | `if (...) { } else { }`  |
| Циклы                   | `while (...)/endwhile` | `while (...) { }`         |
| Строгие блоки `{}`      | Есть (для Java-стиля)| Всегда                    |
| Комментарии             | `'` или `/'*'*'/'`  | `//`                      |
